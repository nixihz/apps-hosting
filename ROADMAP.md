# Keli Apps（apps runtime）Roadmap

Keli Apps 是一个面向团队和开发者的现代、精致、可插拔小应用中枢；底层继续兼容 apps CLI 与运行时。

平台目标是让大家可以用统一规范，把自己的小应用快速接入、部署、运行、更新和下线。应用可以是纯前端，也可以是前端 + 服务端的一体化应用。

---

## 1. 平台定位

**apps** 是一个轻量应用托管与插件化部署平台，适用于：

- 内部工具
- 数据看板
- 运营后台
- AI 小工具
- 查询工具
- 自动化脚本 Web 化
- 团队导航页
- 临时业务应用

支持的应用形态：

1. **纯前端应用**
   - Vue / React / Svelte / 原生 HTML
   - 静态资源部署
   - SPA 路由支持
   - 子路径挂载

2. **前端 + 服务端应用**
   - Node.js / Go / Python / Java 等服务
   - API 服务托管
   - 前后端一体部署
   - 环境变量、端口、健康检查、日志管理

3. **插件型应用**
   - 页面插件
   - 菜单插件
   - API 插件
   - 定时任务插件
   - Webhook 插件

## 2. 核心理念

### 2.1 可插拔

每个应用都是一个独立插件包，可以被安装、启用、停用、升级、卸载。

示例：

```bash
apps install app-demo
apps enable app-demo
apps upgrade app-demo
apps remove app-demo
```

### 2.2 标准化描述

每个应用通过 `apps.yaml` 声明自身信息。

纯前端应用示例：

```yaml
name: demo-dashboard
title: 示例数据看板
type: frontend
version: 1.0.0
entry: dist
route: /x/demo-dashboard
author: team-a
```

前端 + 服务端应用示例：

```yaml
name: todo-app
title: Todo 应用
type: fullstack
version: 1.0.0
route: /x/todo-app

frontend:
  entry: web/dist

backend:
  command: node server/index.js
  port: 3000
  health: /health

env:
  - DATABASE_URL
  - JWT_SECRET
```

### 2.3 平台统一治理

平台提供统一能力：

- 应用注册
- 路由挂载
- 权限控制
- 登录态集成
- 应用市场
- 日志查看
- 健康检查
- 配置管理
- 版本回滚
- 应用审计

## 4. 案例

## 案例一：纯前端应用 —— 团队导航页

### 背景

某团队有很多内部系统链接：

- GitLab
- Jenkins
- Sentry
- 禅道
- 飞书文档
- 测试环境
- 生产监控

以前大家通过收藏夹、群公告、文档维护，容易过期。

### 应用形态

这是一个纯前端应用：

```yaml
name: team-nav
title: 团队导航页
type: frontend
version: 1.0.0
route: /x/team-nav
entry: dist
author: infra-team
```

### 功能

- 系统链接聚合
- 按分类展示
- 支持搜索
- 支持收藏常用链接
- 支持管理员维护链接

### 部署流程

```bash
apps create frontend team-nav
cd team-nav
pnpm build
apps deploy
```

### 用户访问

```text
https://apps.company.com/x/team-nav
```

### 平台价值

- 不需要单独申请服务器
- 不需要单独配置 Nginx
- 不需要每次手工发版
- 平台统一入口、统一权限、统一管理

---

## 案例二：前端 + 服务端应用 —— 临时数据查询工具

### 背景

运营同学经常需要查询一些业务数据，比如：

- 用户订单状态
- 优惠券发放记录
- 活动参与数据
- 退款处理状态

以前需要开发同学手动查数据库，效率低且有安全风险。

### 应用形态

这是一个前端 + 服务端应用：

```yaml
name: data-query-tool
title: 运营数据查询工具
type: fullstack
version: 1.0.0
route: /x/data-query-tool

frontend:
  entry: web/dist

backend:
  command: node server/index.js
  port: 3000
  health: /health

env:
  - DATABASE_URL
  - QUERY_TOKEN

permissions:
  - data-query:view
  - data-query:export
```

### 功能

- 查询订单
- 查询用户
- 查询活动记录
- 导出 CSV
- 操作日志审计
- 根据权限限制查询范围

### 部署流程

```bash
apps create fullstack data-query-tool
cd data-query-tool
pnpm build
apps deploy
```

### 平台提供能力

- 自动启动后端服务
- 管理环境变量
- 暴露访问路径
- 查看服务日志
- 健康检查
- 权限控制
- 失败自动重启

### 用户访问

```text
https://apps.company.com/x/data-query-tool
```

### 平台价值

- 业务工具可以快速上线
- 数据访问有权限控制
- 查询行为可审计
- 开发无需重复搭建登录、部署、日志等基础能力
